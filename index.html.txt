  <section id="overview">
    <h2>Overview</h2>
    <p class="subtitle">
      Understand what Android Studio is and where it fits in the Android development ecosystem.
    </p>
    <p>
      Android Studio is built on IntelliJ IDEA and is the official IDE recommended by Google for
      building Android applications. It integrates the Android SDK, Gradle-based build system,
      device emulators, and a rich set of development tools into a single environment.
    </p>
    <p>
      Whether you are prototyping a small utility app or maintaining a large-scale production
      application, Android Studio provides the tools you need for coding, testing, profiling,
      and publishing Android apps.
    </p>
  </section>

  <section id="features">
    <h2>Key features</h2>
    <p class="subtitle">
      Core capabilities that make Android Studio the central tool for Android developers.
    </p>

    <div class="grid">
      <div class="card">
        <div class="card-title">Intelligent code editor</div>
        <p>
          Code completion, refactoring, and navigation for Kotlin and Java, with deep awareness
          of the Android framework and project structure.
        </p>
        <span class="tag">Productivity</span>
      </div>

      <div class="card">
        <div class="card-title">Layout & UI tools</div>
        <p>
          Visual layout editor, constraint-based design, and preview modes let you design
          responsive interfaces for multiple device sizes.
        </p>
        <span class="tag">UI design</span>
      </div>

      <div class="card">
        <div class="card-title">Build & dependency management</div>
        <p>
          Gradle-based build system that manages variants, dependencies, and signing configurations
          for debug and release builds.
        </p>
        <span class="tag">Build system</span>
      </div>

      <div class="card">
        <div class="card-title">Testing & profiling tools</div>
        <p>
          Integrated profiling for CPU, memory, and network usage, along with support for unit
          tests and instrumented tests on devices and emulators.
        </p>
        <span class="tag">Quality</span>
      </div>
    </div>
  </section>

  <section id="setup">
    <h2>Setup and requirements</h2>
    <p class="subtitle">
      Basic environment requirements and high‑level steps for installing Android Studio.
    </p>

    <p><strong>System requirements (typical desktop):</strong></p>
    <ul class="list">
      <li>Recent 64‑bit operating system (Windows 10 or later, recent macOS, or modern Linux).</li>
      <li>At least 8 GB of RAM is recommended for comfortable development.</li>
      <li>Several gigabytes of free disk space for the IDE, SDK, and emulator images.</li>
      <li>Stable internet connection for downloading SDK components and updates.</li>
    </ul>

    <div class="highlight-box">
      <strong>High‑level installation flow:</strong><br>
      1. Download the latest Android Studio installer from the official Android developers site.<br>
      2. Run the installer and select standard installation to include the Android SDK and virtual device tools.<br>
      3. After installation, open Android Studio and complete the initial setup wizard.<br>
      4. Create a new project to verify that the environment is configured correctly.
    </div>
  </section>

  <section id="workflow">
    <h2>Typical development workflow</h2>
    <p class="subtitle">
      From project creation to testing and preparing builds for release.
    </p>

    <div class="grid">
      <div class="card">
        <div class="card-title">1. Create a project</div>
        <p>
          Start from a template such as “Empty Activity” or “Basic Activity”, define the app name,
          package name, and minimum SDK.
        </p>
      </div>
      <div class="card">
        <div class="card-title">2. Design the UI</div>
        <p>
          Use XML layouts or Jetpack Compose to define screens, following Material Design
          guidelines for consistency and usability.
        </p>
      </div>
      <div class="card">
        <div class="card-title">3. Implement app logic</div>
        <p>
          Write Kotlin or Java code to handle navigation, data, and user interactions,
          organizing code into clear modules and layers.
        </p>
      </div>
      <div class="card">
        <div class="card-title">4. Build, run, and test</div>
        <p>
          Run the app on emulators or physical devices, fix issues reported by the build system,
          and iterate based on testing feedback.
        </p>
      </div>
      <div class="card">
        <div class="card-title">5. Prepare for release</div>
        <p>
          Configure versioning, app signing, and build a release variant suitable for distribution
          through app stores or other channels.
        </p>
      </div>
    </div>
